# insight-agent v0.1.0

## Summary

insight-agent v0.1.0 は、CLI / API の公開入口を整理し、Config 駆動で同じ canonical runner を使うリリースです。既定の返却契約は `output_schema_v2` に揃え、JSON / PDF / text を同じ `InsightRequest` に正規化して処理します。

## Highlights

- 公開 API の本命を `run()` に統一
- CLI の正規入口を `python -m insight_core.cli run ...` に整理
- `RuntimeConfig` による provider / timeout / output / routing 制御を導入
- 既定出力を `output_schema_v2` に統一
- PDF 抽出テキストのキャッシュ再利用を導入
- Claude / Codex 向け repo-local skill を追加

## User-facing changes

### Canonical runner

Python からは次の形が推奨です。

```python
from insight_core import run

result = run(request_dict=payload, config_dict=cfg)
```

### CLI

CLI は `run` サブコマンドを正規入口として扱います。

```bash
python -m insight_core.cli run -i examples/sample_request.json
python -m insight_core.cli run --pdf material/sample.pdf -o output.json
```

旧形式の `python -m insight_core.cli --pdf ...` も後方互換で継続します。

### Output

既定返却は `output_schema_v2` です。`raw` は debug 用に残しています。

## Config precedence

設定の優先順位は次のとおりです。

1. default
2. config file
3. environment variables
4. CLI `--set` / API overrides
5. request-local `config_override`

## Repo organization

このリリースで、補助ファイルの置き場所も整理しました。

- `skills/`: Claude / Codex 向け skill
- `scripts/manual/`: 手動確認スクリプト
- `artifacts/baselines/`: ベンチマーク基準出力
- `artifacts/manual-runs/`: 手動試走の出力

## Validation

リリース前確認:

```bash
python -m pytest -q
python -m insight_core.cli run --help
```

テスト結果は `56 passed`。

## Known limitations

- provider 状態や入力サイズにより長時間実行になることがある
- `raw` は内部確認用であり、外部統合の中心にすべきではない
- 一部の論文固有補正は formatter 側の postprocess で扱っている
